адание 4. SVD

Читаем теорию, можно тут https://education.yandex.ru/handbook/ml/article/matrichnaya-faktorizaciya (особо внимательно “Подход на чистом SVD”) и практические вопросы можно посмотреть тут: https://habr.com/ru/companies/yandex/articles/441586/ Гиперпараметром в этом случае будет размерность разложения, в метриках следует посчитать несколько вариантов.

= Если knn сразу делаете на своих данных (после согласования через гуглформу) - в отчете SVD добавляйте сравнение метрик
= Если knn сделали на фильмах - предварительно примените гибридный подход и посчитайте метрики (отчет отдельно писать не нужно) и после этого в отчете SVD добавляйте сравнение метрик 

Результат = человечески читаемый отчет, начиная с постановки задачи, продолжая описанием структуры и анализом содержания данных,  подхода к реализации, демонстрация на конкретных примерах, описание гиперпараметров модели и как их изменение влияет, расчет метрик качества по всем интересующим нас запускам и интерпретация результатов, выводы (тоже будет гуглоформа, после обсуждения на занятии в нее надо будет загрузить)

Задание 6. Итоговая РС - гибрид нескольких подходов: 

Объединяем решение лучшие решения предыдущих заданий в гибрид. Обязательное условие - использование Content-based подхода. Соответственно, метрики по этой рекомендательной системе должны быть выше, чем по любой предыдущей (гибрид u+i, svd), поэтому нужна сравнительная таблица.

Варианты исполнения: быстрый счет на svd, далее уточнение оценок с помощью u+i и персонализация полученных рекомендаций с помощью content-based; наоборот content-based и уточнение оценок с помощью u+i. Крайне не желательно получить что-то типа простой суммы/бустинга двух моделей, нужно именно последовательное подключение или решение определенной проблемы (холодного старта, разреженности) с помощью подходящей модели.

Результат = человечески читаемый отчет, начиная с постановки задачи, продолжая описанием структуры и анализом содержания данных,  подхода к реализации, демонстрация на конкретных примерах, описание гиперпараметров модели и как их изменение влияет, расчет метрик качества по всем интересующим нас запускам и интерпретация результатов, выводы (гуглформу сделаю к следующей практике, сначала обсудим)
